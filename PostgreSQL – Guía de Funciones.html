TO_DATE()
Convierte un texto en un valor de tipo DATE
Fecha ▶
Sintaxis: TO_DATE(texto, formato)

Toma un string que representa una fecha y lo convierte en un valor DATE real que PostgreSQL puede entender y operar. El formato le dice a PG cómo leer el texto: DD = día, MM = mes, YYYY = año de 4 dígitos.
Ejemplo 1
Conversión básica de texto a fecha
SELECT TO_DATE('13/06/2026', 'DD/MM/YYYY') AS fecha_resultado;
Resultado
fecha_resultado
2026-06-13
Ejemplo 2
Formato con nombre de mes en inglés
SELECT TO_DATE('June 13, 2026', 'Month DD, YYYY') AS fecha;
Resultado
fecha
2026-06-13
Ejemplo 3
Caso de uso real: datos vienen como texto de un formulario
UPDATE clientes SET fecha_nacimiento = TO_DATE(fecha_texto, 'DD-MM-YYYY') WHERE fecha_nacimiento IS NULL;
Si el texto y el formato no coinciden, PG lanza un error. Asegurate de que ambos tengan la misma estructura.
AGE()
Calcula la diferencia entre dos fechas en años, meses y días
Fecha ▶
Sintaxis: AGE(fecha_fin, fecha_inicio) o AGE(fecha)

Devuelve un interval que representa la diferencia entre las dos fechas. Si solo pasás una fecha, la compara con la fecha actual (CURRENT_DATE). Perfecto para calcular edades reales.
Ejemplo 1
Calcular edad de una persona
SELECT AGE('2001-03-15'::DATE) AS edad;
Resultado
edad
25 years 2 mons 29 days
Ejemplo 2
Diferencia entre dos fechas específicas
SELECT AGE('2026-12-31', '2026-01-01') AS diferencia;
Resultado
diferencia
11 mons 30 days
Ejemplo 3
En una tabla real: edad de cada empleado
SELECT nombre, fecha_nacimiento, AGE(fecha_nacimiento) AS edad_actual FROM empleados;
Resultado
nombrefecha_nacimientoedad_actual
Ana López1990-05-2035 years 0 mons 24 days
Carlos Mora1985-11-0340 years 7 mons 10 days
Para extraer solo el número de años de AGE(), combinalo con EXTRACT: EXTRACT(YEAR FROM AGE(fecha_nacimiento))
EXTRACT()
Extrae una parte específica de una fecha o intervalo
Fecha ▶
Sintaxis: EXTRACT(campo FROM fecha)

Campos disponibles: YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, DOW (día de semana 0=domingo), DOY (día del año), WEEK, QUARTER. Devuelve un número.
Ejemplo 1
Extraer partes básicas
SELECT EXTRACT(YEAR FROM CURRENT_DATE) AS anio, EXTRACT(MONTH FROM CURRENT_DATE) AS mes, EXTRACT(DAY FROM CURRENT_DATE) AS dia;
Resultado
aniomesdia
2026613
Ejemplo 2
Ventas por mes del año
SELECT EXTRACT(MONTH FROM fecha_venta) AS mes, SUM(total) AS ventas_mes FROM ventas GROUP BY mes ORDER BY mes;
Resultado
mesventas_mes
158000.00
243200.00
371500.00
Ejemplo 3
Edad en años solamente (combinando con AGE)
SELECT nombre, EXTRACT(YEAR FROM AGE(fecha_nacimiento)) AS edad_anos FROM empleados;
Resultado
nombreedad_anos
Ana López35
Carlos Mora40
DOW: 0 = domingo, 1 = lunes ... 6 = sábado. Útil para filtrar registros de un día específico de la semana.
Funciones de Agregación
SUM, AVG, COUNT, MAX, MIN — resumen grupos de filas
Agregación ▶
Las funciones de agregación toman un conjunto de filas y devuelven un único valor. Siempre van acompañadas de GROUP BY cuando querés agrupar, o solas si querés el total de toda la tabla.

COUNT(col) – cuenta filas no nulas · SUM(col) – suma · AVG(col) – promedio · MAX(col) – mayor valor · MIN(col) – menor valor
Ejemplo 1
Totales simples de toda la tabla
SELECT COUNT(*) AS total_empleados, AVG(salario) AS salario_promedio, MAX(salario) AS salario_maximo, MIN(salario) AS salario_minimo, SUM(salario) AS planilla_total FROM empleados;
Resultado
totalpromediomaximominimoplanilla
48750000.002400000.00420000.0036000000.00
Ejemplo 2
Agregación con GROUP BY — salario promedio por departamento
SELECT departamento, COUNT(*) AS cantidad, ROUND(AVG(salario), 2) AS promedio FROM empleados GROUP BY departamento ORDER BY promedio DESC;
Resultado
departamentocantidadpromedio
IT121200000.00
Ventas20850000.00
RRHH8680000.00
Ejemplo 3
HAVING — filtrar grupos (equivalente de WHERE para agregaciones)
SELECT departamento, COUNT(*) AS cant, SUM(salario) AS total_salarios FROM empleados GROUP BY departamento HAVING COUNT(*) > 5 -- solo depto con más de 5 empleados ORDER BY total_salarios DESC;
Regla de oro: si está en SELECT y NO es una función de agregación, tiene que estar en GROUP BY.
TO_CHAR()
Convierte fechas y números a texto con formato personalizado
Conversión ▶
Sintaxis: TO_CHAR(valor, formato)

Es el inverso de TO_DATE. Toma una fecha o número y lo convierte en texto con el formato que vos especificás. Muy útil para reportes y visualización.

Formatos de fecha: YYYY, MM, DD, HH24, MI, SS, Month, Day. Formatos de número: 999, FM, G (separador miles), D (decimal), L (símbolo moneda local).
Ejemplo 1
Formatear una fecha para mostrarla al usuario
SELECT TO_CHAR(CURRENT_DATE, 'DD "de" Month "de" YYYY') AS fecha_legible;
Resultado
fecha_legible
13 de June de 2026
Ejemplo 2
Formatear números — salario con símbolo y separadores
SELECT nombre, TO_CHAR(salario, 'L999G999G999D00') AS salario_formateado FROM empleados;
Resultado
nombresalario_formateado
Ana López₡ 750,000.00
Carlos Mora₡ 1,200,000.00
Ejemplo 3
Generar un periodo tipo "2026-06" para agrupar por mes
SELECT TO_CHAR(fecha_venta, 'YYYY-MM') AS periodo, SUM(total) AS ventas FROM ventas GROUP BY periodo ORDER BY periodo;
Los textos dentro del formato van entre comillas dobles: 'DD "de" Month'. FMMonth quita los espacios extra del nombre del mes.
CAST() y ::
Convierte un valor de un tipo de dato a otro
Conversión ▶
Sintaxis larga: CAST(valor AS tipo)
Sintaxis corta: valor::tipo

Ambas hacen lo mismo: convertir un valor a otro tipo. La versión :: es exclusiva de PostgreSQL y es la más usada por ser más corta. Tipos comunes: INTEGER, NUMERIC, TEXT, DATE, TIMESTAMP, BOOLEAN, VARCHAR(n).
Ejemplo 1
Texto a número para operar
SELECT '150'::INTEGER + 50 AS resultado, CAST('3.14' AS NUMERIC) * 2 AS pi_doble;
Resultado
resultadopi_doble
2006.28
Ejemplo 2
Número a texto para concatenar
SELECT 'Empleado #' || id::TEXT AS etiqueta, nombre || ' tiene ' || edad::TEXT || ' años' AS descripcion FROM empleados;
Resultado
etiquetadescripcion
Empleado #1Ana López tiene 35 años
Empleado #2Carlos Mora tiene 40 años
Ejemplo 3
Texto a fecha (equivalente a TO_DATE simple)
SELECT '2026-06-13'::DATE AS fecha, '2026-06-13 14:30:00'::TIMESTAMP AS timestamp_val;
:: es syntactic sugar de PG — más corto y más legible. Para código portable entre motores SQL, usá CAST().
CONCAT() y ||
Une textos en un solo string
Texto ▶
Diferencia clave:
|| — Si algún valor es NULL, el resultado completo es NULL.
CONCAT() — Ignora los NULL, los trata como string vacío.

Podés usar ambos según si querés que el NULL "infecte" el resultado o no.
Ejemplo 1
Concatenar nombre completo
SELECT nombre || ' ' || apellido AS nombre_pipe, CONCAT(nombre, ' ', apellido) AS nombre_concat FROM empleados;
Resultado
nombre_pipenombre_concat
Ana LópezAna López
Ejemplo 2
Diferencia con NULL (importante)
SELECT 'Hola ' || NULL || ' mundo' AS con_pipe, -- → NULL CONCAT('Hola ', NULL, ' mundo') AS con_concat; -- → 'Hola mundo'
Resultado
con_pipecon_concat
NULLHola mundo
Ejemplo 3
CONCAT_WS — concatenar con separador (muy útil)
SELECT CONCAT_WS(', ', nombre, apellido, ciudad) AS datos FROM clientes; -- CONCAT_WS ignora NULLs Y agrega el separador automáticamente
Resultado
datos
Ana, López, San José
Carlos, Mora, Limón
CONCAT_WS (With Separator) es frecuentemente la mejor opción — pone el separador solo entre los valores no-nulos.
SUBSTRING()
Extrae una porción de un texto
Texto ▶
Sintaxis: SUBSTRING(texto FROM inicio FOR longitud) o SUBSTRING(texto, inicio, longitud)

Extrae una subcadena empezando en la posición inicio (los índices empiezan en 1, no en 0) y tomando longitud caracteres. Si omitís longitud, toma hasta el final.
Ejemplo 1
Extraer las primeras letras
SELECT SUBSTRING('Informática Empresarial', 1, 11) AS primera_palabra;
Resultado
primera_palabra
Informática
Ejemplo 2
Extraer el año de un código formateado
SELECT codigo, SUBSTRING(codigo, 1, 4) AS anio, SUBSTRING(codigo, 6) AS numero FROM facturas; -- Ejemplo de código: '2026-00842'
Resultado
codigoanionumero
2026-00842202600842
Ejemplo 3
Usar con POSITION para encontrar dónde cortar
SELECT email, SUBSTRING(email, 1, POSITION('@' IN email) - 1) AS usuario FROM usuarios;
Resultado
emailusuario
josse@ucr.ac.crjosse
En PG también existe LEFT(texto, n) y RIGHT(texto, n) para extraer desde los extremos de forma más legible.
LENGTH()
Devuelve la longitud (cantidad de caracteres) de un texto
Texto ▶
Sintaxis: LENGTH(texto) o CHAR_LENGTH(texto)

Devuelve el número de caracteres del string. NULL si el valor es NULL. Cuenta caracteres Unicode, no bytes (importante para textos con acentos o ñ).
Ejemplo 1
Longitud básica
SELECT LENGTH('Hola mundo') AS lon_hola, LENGTH('Limón') AS lon_limon;
Resultado
lon_holalon_limon
105
Ejemplo 2
Validar que contraseñas tengan mínimo 8 caracteres
SELECT usuario, LENGTH(contrasena) AS longitud, CASE WHEN LENGTH(contrasena) >= 8 THEN 'OK' ELSE 'Muy corta' END AS estado FROM usuarios;
Resultado
usuariolongitudestado
josse12OK
user24Muy corta
Ejemplo 3
Filtrar por longitud
SELECT * FROM productos WHERE LENGTH(descripcion) > 200; -- descripciones largas
TRIM()
Elimina espacios (u otros caracteres) del inicio, fin o ambos lados
Texto ▶
Variantes:
TRIM(texto) — elimina espacios de ambos lados
LTRIM(texto) — solo del lado izquierdo
RTRIM(texto) — solo del lado derecho
TRIM(LEADING 'x' FROM texto) — elimina el caracter 'x' del inicio
TRIM(BOTH 'x' FROM texto) — elimina 'x' de ambos lados
Ejemplo 1
Limpiar espacios básico
SELECT TRIM(' Hola mundo ') AS limpio, LTRIM(' Hola mundo ') AS solo_izquierda, RTRIM(' Hola mundo ') AS solo_derecha;
Resultado
limpiosolo_izquierdasolo_derecha
'Hola mundo''Hola mundo '' Hola mundo'
Ejemplo 2
Limpiar datos sucios de una importación
UPDATE clientes SET nombre = TRIM(nombre), apellido = TRIM(apellido), email = LOWER(TRIM(email));
Ejemplo 3
Eliminar caracteres específicos (no solo espacios)
SELECT TRIM(BOTH '$' FROM '$$1500$$') AS precio_limpio; -- → '1500'
Siempre que recibís datos de formularios o archivos CSV, pasalos por TRIM + LOWER para normalizar antes de comparar.
NULLIF()
Devuelve NULL si dos valores son iguales, sino devuelve el primero
Lógica ▶
Sintaxis: NULLIF(valor1, valor2)

Si valor1 = valor2 → devuelve NULL.
Si valor1 ≠ valor2 → devuelve valor1.

Su uso más común: evitar divisiones por cero, o convertir un valor "vacío" como '' o 0 en NULL real para que otras funciones lo ignoren.
Ejemplo 1
Evitar división por cero (clásico)
SELECT ventas / NULLIF(dias_laborales, 0) AS ventas_por_dia FROM reporte; -- Si dias_laborales = 0 → resultado es NULL, no un error
Resultado
ventas_por_dia
2500.00
NULL
Ejemplo 2
Tratar strings vacíos como NULL
SELECT nombre, NULLIF(telefono, '') AS telefono_limpio FROM clientes; -- '' se convierte en NULL real
Resultado
nombretelefono_limpio
Ana8888-4444
CarlosNULL
Ejemplo 3
Combinado con COALESCE
SELECT COALESCE(NULLIF(descuento, 0), precio * 0.10) AS descuento_efectivo FROM pedidos; -- Si descuento = 0 → usa el 10% del precio como fallback
NULLIF + COALESCE es una combinación muy poderosa para manejar valores "vacíos" de distintos tipos.
CASE
Estructura condicional dentro de una consulta SQL
Lógica ▶
Hay dos formas:

CASE simple — compara un valor con varios posibles:
CASE columna WHEN val1 THEN res1 WHEN val2 THEN res2 ELSE resN END

CASE buscado — evalúa condiciones booleanas independientes:
CASE WHEN cond1 THEN res1 WHEN cond2 THEN res2 ELSE resN END
Ejemplo 1
Categorizar salarios en rangos
SELECT nombre, salario, CASE WHEN salario < 500000 THEN 'Bajo' WHEN salario < 1000000 THEN 'Medio' WHEN salario < 2000000 THEN 'Alto' ELSE 'Ejecutivo' END AS rango_salarial FROM empleados;
Resultado
nombresalariorango_salarial
Ana López750000Medio
Carlos Mora1200000Alto
Luis Pérez2800000Ejecutivo
Ejemplo 2
CASE simple — traducir un código
SELECT pedido_id, CASE estado WHEN 'P' THEN 'Pendiente' WHEN 'E' THEN 'En camino' WHEN 'C' THEN 'Completado' ELSE 'Desconocido' END AS estado_legible FROM pedidos;
Ejemplo 3
CASE dentro de COUNT — contar condicionalmente
SELECT departamento, COUNT(*) AS total, COUNT(CASE WHEN salario > 1000000 THEN 1 END) AS senior FROM empleados GROUP BY departamento;
Resultado
departamentototalsenior
IT128
Ventas205
CASE siempre termina con END. ELSE es opcional; si no ponés ELSE y ninguna condición aplica, devuelve NULL.
Subconsultas (Subqueries)
Una consulta dentro de otra consulta
Avanzado ▶
Una subconsulta es un SELECT dentro de otro SELECT, WHERE, HAVING o FROM. Se ejecuta primero y su resultado es usado por la consulta exterior. Pueden ser:

• Escalares — devuelven un solo valor
• De fila — devuelven una fila
• De tabla — devuelven múltiples filas (usadas con IN, ANY, ALL, EXISTS)
Ejemplo 1
Subconsulta escalar en WHERE — empleados con salario mayor al promedio
SELECT nombre, salario FROM empleados WHERE salario > ( SELECT AVG(salario) FROM empleados );
Resultado — la subconsulta calcula primero: AVG = 750000
nombresalario
Carlos Mora1200000
Luis Pérez2800000
Ejemplo 2
Subconsulta con IN — clientes que tienen pedidos
SELECT nombre, email FROM clientes WHERE id IN ( SELECT DISTINCT cliente_id FROM pedidos WHERE total > 10000 );
Ejemplo 3
Subconsulta en FROM (tabla derivada)
SELECT depto, promedio FROM ( SELECT departamento AS depto, ROUND(AVG(salario), 2) AS promedio FROM empleados GROUP BY departamento ) AS promedios_depto WHERE promedio > 800000;
Resultado
deptopromedio
IT1200000.00
Gerencia2100000.00
Para subconsultas complejas o reutilizables, considerá usar WITH (ver sección siguiente) — es más legible.
WITH (CTE — Common Table Expressions)
Define subconsultas con nombre reutilizables
Avanzado ▶
Sintaxis: WITH nombre_cte AS (SELECT ...) SELECT ... FROM nombre_cte

Un CTE es como darle un nombre a una subconsulta para poder referenciarla varias veces en la misma query. Hace el código mucho más legible, especialmente cuando tenés lógica compleja o necesitás el mismo resultado en varios lugares.
Ejemplo 1
CTE básico — equivalente limpio a la subconsulta en FROM
WITH promedios_depto AS ( SELECT departamento, ROUND(AVG(salario), 2) AS promedio FROM empleados GROUP BY departamento ) SELECT * FROM promedios_depto WHERE promedio > 800000;
Resultado
departamentopromedio
IT1200000.00
Gerencia2100000.00
Ejemplo 2
Múltiples CTEs encadenados
WITH ventas_mes AS ( SELECT TO_CHAR(fecha, 'YYYY-MM') AS mes, SUM(total) AS total_mes FROM ventas GROUP BY mes ), promedio_general AS ( SELECT AVG(total_mes) AS prom FROM ventas_mes ) SELECT vm.mes, vm.total_mes, pg.prom AS promedio_global, vm.total_mes - pg.prom AS diferencia FROM ventas_mes vm, promedio_general pg ORDER BY vm.mes;
Ejemplo 3
Reutilizar el CTE en múltiples partes de la query
WITH top_clientes AS ( SELECT cliente_id, SUM(total) AS total_compras FROM pedidos GROUP BY cliente_id ORDER BY total_compras DESC LIMIT 10 ) SELECT c.nombre, tc.total_compras, 'VIP' AS categoria FROM top_clientes tc JOIN clientes c ON c.id = tc.cliente_id;
WITH hace que queries complejas sean mucho más legibles. En lugar de subconsultas anidadas, definís cada paso con un nombre descriptivo.
FLOOR() / CEIL() / ROUND()
Redondeo hacia abajo, hacia arriba, o al más cercano
Matemática ▶
FLOOR(n) — redondea hacia abajo (trunca al entero inferior).
CEIL(n) — redondea hacia arriba (entero superior).
ROUND(n, decimales) — redondea al número de decimales indicado. Si omitís decimales, redondea al entero más cercano.

El uso estrella de FLOOR en bases de datos: calcular décadas a partir de un año. Dividís el año entre 10, aplicás FLOOR, y multiplicás por 10.
Ejemplo 1
Comportamiento básico de los tres
SELECT FLOOR(7.9) AS floor_val, -- → 7 CEIL(7.1) AS ceil_val, -- → 8 ROUND(7.5) AS round_val, -- → 8 ROUND(7.456, 2) AS round_dec; -- → 7.46
Resultado
floor_valceil_valround_valround_dec
7887.46
Ejemplo 2
Agrupar ventas por década — la fórmula clásica de FLOOR
SELECT FLOOR(EXTRACT(YEAR FROM fecha_venta) / 10) * 10 AS decada, COUNT(*) AS total_ventas, SUM(monto) AS ingresos FROM ventas GROUP BY decada ORDER BY decada; -- 1987 → FLOOR(198.7) * 10 = 1980 | 2003 → FLOOR(200.3) * 10 = 2000
Resultado
decadatotal_ventasingresos
20001240380000.00
201048301920000.00
202081004150000.00
Ejemplo 3
CEIL para calcular páginas necesarias (paginación)
SELECT CEIL(COUNT(*) / 10.0) AS total_paginas FROM productos; -- 45 productos / 10 por página = CEIL(4.5) = 5 páginas
Resultado
total_paginas
5
La fórmula de décadas FLOOR(año / 10) * 10 funciona para cualquier agrupación periódica: décadas, lustros (/ 5), siglos (/ 100). CEIL es ideal para paginación.
COALESCE()
Devuelve el primer valor no-nulo de una lista
Lógica ▶
Sintaxis: COALESCE(val1, val2, val3, ...)

Evalúa los argumentos en orden y devuelve el primero que no sea NULL. Si todos son NULL, devuelve NULL. Es el mecanismo estándar para definir valores por defecto cuando un campo puede ser nulo.
Ejemplo 1
Valor de respaldo si el campo es NULL
SELECT nombre, COALESCE(telefono, celular, 'Sin contacto') AS contacto FROM clientes; -- Si telefono es NULL, prueba celular. Si también NULL, muestra texto fijo
Resultado
nombrecontacto
Ana López8888-4444
Carlos Mora7777-3333
Luis PérezSin contacto
Ejemplo 2
Evitar NULLs en concatenación — problema clásico con ||
SELECT COALESCE(nombre, '') || ' ' || COALESCE(apellido1, '') || ' ' || COALESCE(apellido2, '') AS nombre_completo FROM personas; -- Sin COALESCE, si apellido2 es NULL el resultado completo sería NULL
Resultado
nombre_completo
Ana López Herrera
Carlos Mora
Ejemplo 3
COALESCE + NULLIF — combo para limpiar strings vacíos
SELECT id, COALESCE( NULLIF(TRIM(direccion), ''), 'Sin dirección registrada' ) AS direccion_limpia FROM clientes; -- TRIM quita espacios → NULLIF convierte '' en NULL → COALESCE pone el fallback
Resultado
iddireccion_limpia
1Av. Central, San José
2Sin dirección registrada
COALESCE es la función de "fallback". NULLIF convierte valores vacíos en NULL. Usados juntos son la dupla perfecta para limpiar datos sucios de sistemas heredados.
UPPER() / LOWER()
Convierte texto a mayúsculas o minúsculas
Texto ▶
UPPER(texto) — convierte todo a MAYÚSCULAS.
LOWER(texto) — convierte todo a minúsculas.
INITCAP(texto) — Primera Letra De Cada Palabra En Mayúscula.

Esenciales para comparaciones insensibles a mayúsculas y para normalizar datos antes de almacenarlos o compararlos.
Ejemplo 1
Comparación insensible a mayúsculas — el caso típico
SELECT * FROM productos WHERE UPPER(TRIM(categoria)) = 'ELECTRONICA'; -- Encuentra 'Electronica', 'ELECTRONICA', 'electronica', ' Electronica '
Ejemplo 2
Normalizar emails antes de guardar
UPDATE usuarios SET email = LOWER(TRIM(email)); -- 'Usuario@Gmail.COM ' → 'usuario@gmail.com' -- También útil al comparar sin modificar la tabla: SELECT * FROM usuarios WHERE LOWER(email) = LOWER('Usuario@Gmail.com');
Ejemplo 3
INITCAP para presentar nombres con formato bonito
SELECT id, INITCAP(LOWER(nombre)) AS nombre, INITCAP(LOWER(apellido)) AS apellido FROM empleados; -- 'MARIA JOSE' → 'Maria Jose' | 'juan' → 'Juan'
Resultado
idnombreapellido
1Maria JoseVargas
2LuisJiménez
Regla de oro: guardá los datos normalizados (LOWER para emails, UPPER o INITCAP para nombres) y comparalos siempre en el mismo caso. Evita duplicados "invisibles".
JOINs (INNER, LEFT, SELF)
Combina filas de dos o más tablas según una condición
JOINs ▶
Un JOIN une tablas por una columna en común. Los tipos principales:

INNER JOIN — solo filas que tienen coincidencia en AMBAS tablas.
LEFT JOIN — TODAS las filas de la tabla izquierda + las que coincidan de la derecha (NULL si no hay match).
RIGHT JOIN — lo inverso del LEFT.
FULL OUTER JOIN — todas las filas de ambas tablas, con NULL donde no haya match.
SELF JOIN — una tabla unida consigo misma (para relaciones jerárquicas).
Ejemplo 1 — INNER JOIN
Consolidado demográfico: ciudadanos con su descripción de sexo
SELECT s."Des_Sexo", COUNT(c."TC_Cedula") AS total_ciudadanos FROM public."T_Sexo" s INNER JOIN public."Costarricense" c ON s."Cod_sexo" = c."TN_Sexo"::TEXT -- cast necesario: bigint → text GROUP BY s."Des_Sexo"; -- Solo muestra sexos que tienen al menos 1 ciudadano
Resultado
Des_Sexototal_ciudadanos
Masculino2481034
Femenino2519823
Ejemplo 2 — LEFT JOIN
Mismo reporte pero incluye sexos SIN ciudadanos (práctica 3.3)
SELECT s."Des_Sexo", COUNT(c."TC_Cedula") AS total_ciudadanos FROM public."T_Sexo" s LEFT JOIN public."Costarricense" c ON s."Cod_sexo" = c."TN_Sexo"::TEXT GROUP BY s."Des_Sexo" ORDER BY total_ciudadanos DESC; -- Aparecen TODOS los registros de T_Sexo, con 0 si no tienen ciudadanos
Resultado — nótese el sexo sin registros
Des_Sexototal_ciudadanos
Masculino2481034
Femenino2519823
No binario0
Ejemplo 3 — SELF JOIN
Ciudadano con datos de su madre (práctica 3.6)
SELECT hijo."TC_Cedula" AS cedula_hijo, CONCAT(hijo."TC_Nombre", ' ', hijo."TC_Apellido1") AS nombre_hijo, madre."TC_Cedula" AS cedula_madre, CONCAT(madre."TC_Nombre", ' ', madre."TC_Apellido1") AS nombre_madre FROM public."Costarricense" hijo INNER JOIN public."Costarricense" madre ON hijo."TC_Cedula_Madre" = madre."TC_Cedula"; -- La misma tabla aparece dos veces con alias distintos: hijo y madre
Resultado
cedula_hijonombre_hijocedula_madrenombre_madre
101230456Carlos Morales202110789María Castro
301440122Ana González501880345Rosa González
La regla para elegir JOIN: ¿necesitás las filas sin match? LEFT JOIN. ¿Solo las que coinciden? INNER JOIN. ¿La tabla se relaciona con ella misma? SELF JOIN con dos alias distintos.
HAVING vs WHERE
Cuándo filtrar antes de agrupar y cuándo después
Agregación ▶
WHERE filtra filas antes de que se forme el grupo — no puede usar funciones de agregación.
HAVING filtra después de agrupar — puede usar SUM, COUNT, AVG, etc.

Orden de ejecución real en SQL: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
Ejemplo 1
WHERE filtra filas individuales antes de agrupar
SELECT TC_Lugar_Nacimiento, COUNT(*) AS total FROM public."Costarricense" WHERE TN_Fallecido = 0 -- filtra antes: solo vivos GROUP BY TC_Lugar_Nacimiento;
Ejemplo 2
HAVING filtra grupos — lugares con más de 5000 nacimientos (práctica 3.7)
SELECT TC_Lugar_Nacimiento AS lugar, COUNT(*) AS total_nacimientos FROM public."Costarricense" WHERE TC_Lugar_Nacimiento IS NOT NULL AND TRIM(TC_Lugar_Nacimiento) <> '' GROUP BY TC_Lugar_Nacimiento HAVING COUNT(*) > 5000 -- filtra grupos después de agrupar ORDER BY total_nacimientos DESC;

Ejemplo 3
Combinando WHERE y HAVING en la misma query
SELECT departamento, COUNT(*) AS empleados_activos, ROUND(AVG(salario), 0) AS salario_prom FROM empleados WHERE activo = TRUE -- solo activos (filtra filas) GROUP BY departamento HAVING AVG(salario) > 800000 -- solo deptos con buen promedio (filtra grupos) ORDER BY salario_prom DESC;
Truco mental: WHERE = "¿qué filas entran al grupo?" / HAVING = "¿qué grupos sobreviven?" No podés poner COUNT(*) en WHERE — eso revienta la query.
